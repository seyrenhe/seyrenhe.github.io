archlinux从wicd转到netctl

High一下
2015-11-18 · · 技术 ·

问题

想把笔记本上跑的archlinux的无线管理方式从wicd-gtk转为netctl,用systemd来管理,折腾过程中出现很多问题

解决过程

  1. 搜索archlinux netctl 
在搜索结果中参考[依云大大的文章Arch Linux 自动连接可用无线网络](http://lilydjwg.is-programmer.com/2014/9/2/auto-connecting-to-wi-fi-in-arch-linux.59235.html)


按照文章里说的先把/etc/netctl/examples里的配置文件拷出来.这里我用的wpa加密,所以我拷的是wpa那个配置文件,放到/etc/netctl/目录下.命名任意,我命名成home了.然后注意一定要把那个文件的属性改成600,禁止非root用户访问,不改的话,netctl会告警的,启动不了.因为配置文件里有写明文的wifi密码.


然后编辑配置文件

摘录一段依云文章里的说明

之所以要先更改权限再编辑，是因为某些编辑器（如 Vim）会生成同权限的备份文件；那里有可能也会包含密码。可以放多份配置文件在这里，netctl-auto 默认会去找一个可用的连接。有多个可用的时候不太清楚它会连上哪一个，可以使用更复杂的配置文件来指定优先级（参见 examples/wireless-wpa-configsection 示例配置）。

配置文件写好之后,就可以用systemd来启动了.

同样摘录一段依云文章里的说明

Arch Linux 一贯的传统是不启动不必要的服务，除非用户说要启动之。netctl-auto 的 systemd 服务名是 netctl-auto@interface.service（当然 .service 后缀还是可以省略的）。interface 部分写你的无线网络接口的名字，可以通过 ip link、ifconfig、iwconfig 等命令看到。我禁用了 systemd 的http://www.freedesktop.org/wiki/Software/systemd/PredictableNetworkInterfaceNames/，所以我的无线网络接口名唤 wlan0。我使用如下命令启动服务：
$ sudo systemctl start netctl-auto@wlan0.service

当然一切顺利的话,就能连上网了,不过我折腾的时候碰到了好几个问题.

  1. 问题1:提示interface已被占用.解决方法:因为wicd-gtk的存在,所以冲突了.卸载wicd-gtk然后重启即可.注意用pacman -Rs wicd-gtk remove wicd-gtk的时候,会把wpa-supplicant包也给remove了.这个netctl连wpa无线网络验证的时候要用到,所以要用pacman -S wpa-supplicant重新装一下.
  2. 问题2:提示wpa-actiond: command not found. 解决方法:用pacman -S wpa_actiond安装wpa-actiond包

上面问题解决好了就可以用systemctl enable netctl-auto@wlan0.service 设置开机启动了

另外可以通过netctl-auto list命令查看连接上了哪个配置文件里指定的热点

总结

要学会查日志

附上systemd模块问题查找的几个常用命令

systemctl status netctl-auto@wlp3s0.service 查看netctl模块的状态

journalctl -u netctl-auto@wlp3s0.service 查看netctl模块的日志
