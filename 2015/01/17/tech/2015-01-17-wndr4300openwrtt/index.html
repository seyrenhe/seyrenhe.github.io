wndr4300刷openwrt

High一下
2015-01-17 · · 技术 ·

最近入了一个wndr4300的路由器，很高大上，据说很容易折腾，于是为了抱着“生命不体，折腾不止”的精神，开始在网上搜索wndr4300的相关内容。经过一段时间的搜索，还真让我找到了不好折腾方法，其中最实用的当属，刷openwrt了.openwrt可以看作是一个跑在路由器上的小型linux系统，并且可以在系统的基础上定制很多功能，比如科学上网，远程下载等等。

###准备工作###

  1. wndr4300一台。。。这里最基本的
  2. openwrt 包一个，可以从这里下载适配4300的openwrt包
  3. 编译好的shadowsocks chinadns ssgui chinadnsgui 百度盘
  4. 能用来连openwrt的ssh软件一个，和我一样用linux的就不用了准备了。

###刷机###

  1. 登录wndr4300后台。网址为192.168.1.1
  2. 在高级-升级那里选择刚下好的openwrt img包，等它上传完，会现实版本。按确定后，路由器开始更新系统，等会就会重启。

  3. 重启完毕后， 拿根网上，连上路由器的lan口，设置系统ip为192.168.1.2 掩码为 255.255.255.0 网关为192.168.1.1 archlinux可以和我一样用以下命令设置： 

    $ sudo ip addr add 192.168.1.2/24 broadcast 192.168.1.255 dev 有线网卡名 #这是设置Ip
$ sudo ip route add default via 192.168.1.1 #这是设置网关
  4. 设置好后，再登录192.168.1.1,你会发现界面变成openwrt的界面了。首次登录设置一下密码，后面ssh登录也要用，openwrt的设置请自行google

  5. 我们再来说下安装shadowsocks和chinadns,把上面baidu盘里的东西解压出来，然后通过scp命令上传到路由器的/tmp目录下，具体命令为：

    scp 解压好的目录/*.ipk root@192.168.1.1:/tmp

  6. 通过ssh登录路由器来安装：

    $ssh root@192.168.1.1
$cd /tmp;opkg install *.ipk

  7. 如果出现依赖错误，例如libopenssl,就执行下面命令：

    $opkg update #首次登录要更新一下包数据库
$opkg install libopenssl
解决依赖后再执行第6步里的 
$opkg install *.ipk

  8. 接下后修改dnsmasq

    $vim /etc/dnsmasq.conf
最后加上
no-resolv
server=127.0.0.1#5353
  9. 至此基本参数配置完毕，接下来在openwrt web管理页面里配置一下ss和chinadns就可以了

###需要注意的地方###

  1. 如果你在上面开启了udp转发功能，那么你需要你的shadowsocks支持udp转发功能，shadowsocks-python版不支持，需要把server端改成node.js版或者c的shadowsocks-libev版，并且启动的时候要加-u参数，注：debian7通过apt安装的shadowsocks-libev版默认已开启udp转发。

###【转】介绍一下chinadns的作用###

方案基于shadowsocks-libev的ss-redir功能，根据网上获取的一个路由表作为判定条件，将访问国外IP的流量转发至代理服务器。 不过ss-redir只支持TCP的转发，UDP的是不行的，因此要解决DNS问题，因为DNS默认是UDP。

要解决DNS解析问题，确保国内站点利用国内DNS解析，国外站点转发至代理服务器解析。显然在客户端完美实现比较困难，于是乎有个退而求其次的判断方法，同时对国内DNS和国外DNS（走代理）发起DNS请求，根据返回的IP地址判断采用哪个的结果（也就是我们用到的ChinaDNS用的方法）：

1、国内DNS返回结果为国内IP，国外DNS返回结果同样为国内IP或者返回失败，则采用国内DNS解析结果；

2、国外DNS返回结果为国外IP，国内DNS返回结果同样为国外IP或者返回失败，则采用国外DNS解析结果；

3、国内DNS返回国内IP，国外DNS返回国外IP，则如果国内IP不在黑名单列表（GFW的DNS污染IP黑名单）中，采用国内IP，否则采用国外IP；

4、不符合上述结果的，返回失败。

另外对于某些客户端软件可能会使用UDP方式进行一些通讯，比如说外服游戏的客户端，这类软件，如果是PC上你还是老老实实用Proxifier之类的软件转发至本地的shadowsocks客户端吧，如果是手机上还是要老老实实开shadowsocks软件，并且要使用VPN方式。不过这类比较非常少见，大部分情况，被墙的服务还是采用TCP的。

####chinadns就是能够解决dns污染实现dns，实现udp转发的一个工具####
