如何配置一台新vps

High一下
2015-01-21 · · 技术 ·

最近一台digitalocean的vps越来越不给力了，于是和沈老师商量了一下，想和他一起租一台linode的vps，玩vps的人都应该知道linode，那是美国一家土豪vps运营商，以前最便宜的套餐也要几十美元，不过最近推出了屌丝套餐，只要10美元就够了，10美元，你买不了吃亏，10美元，你买不了上当。。。。。。。。（滚租）

于是很快一台centos 系统的vps就弄好了，不过这也只是一台什么都没有的机子而已，我们要给他配置好应用。维护好系统的安全。总结一下，有以下步骤：

  • bash换成zsh
  • ssh验证方式改成公钥验证，关毕root的密码登录
  • lnmp环境安装（待补全）

###bash换成zsh###

  1. 安装zsh: 

    yum install zsh

  2. 切换到zsh:

    chsh -s /bin/zsh 或者 usermod -s /bin/zsh root

  3. 配置zsh,推荐用oh-my-zsh

    如果是新的vps，你得先安装git: yum install git
git clone git://github.com/robbyrussell/oh-my-zsh.git ~/.oh-my-zsh

    cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

###ssh验证方式改成公钥验证###

  1. 在你使用的系统上运行以下命令来生成公私密钥对:

    ssh-keygen 
一直按回车默认就行，

    然后你会发现~/.ssh/下有了一个id_rsa私钥文件和一个id_rsa.pub的公钥文件

  2. 上传公钥到vps：
    先在vps上用ls -al查看一下.ssh目录是否存在。
    如果不存在需要用mkdir .ssh创建这个目录。注意.ssh目录的权限最好为700,authorized_keys文件的权限最好为600
    如果存在就可以运行以下命令。

    scp ~/.ssh/id_rsa.pub user@example.com:/home/user/.ssh/uploaded_key.pub

  3. 把上传的公钥设置为验证密钥

    ssh user@example.com "echo `cat ~/.ssh/uploaded_key.pub` >> ~/.ssh/authorized_keys"

  4. 关闭密码登录
    编辑/etc/ssh/sshd_config
    找到#PasswordAuthentication yes
    改成PasswordAuthentication no

  5. 更改ssh监听端口
    编辑/etc/ssh/sshd_config
    把port 22改成你相要的端口,最好是4位以上的端口
    重启ssh服务
    centos7用：

    systemctl  restart sshd

    这样我们就完成了用公钥登录vps，这样别人就算能猜到密码也登录不了，你的私钥一定要保存好，这个丟了的话，你自已也登录不了，只能通过vps管理系统的后台console修改登录方式了。
