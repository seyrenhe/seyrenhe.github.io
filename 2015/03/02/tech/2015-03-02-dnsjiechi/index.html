###论家庭路由器wndr4300(openwrt)实行dns劫持的可行性###
如果你不想让家里人上一些网址，比如360啊，115啊4399小游戏啊什么的，你可以学习一个广大isp商的手段，那就是dns劫持。

简单的说就是强行让家里的所有终端使用你的dns，然后把指定url的ip指向你想让他们访问的其它网站的ip.

  1. 你需要安装dnsmasq

  2. 在dnsmasq.con里加入如下配置：

    address=/根域名/你想要让访问那个域名的人访问的ip

比如 address=/baidu.com/216.58.216.36 这样访问xxx.baidu.com的请求都会被发送到后面那个ip.

而那个ip是谷歌，实现的效果就是访问baidu都会被专跳到google

###当git server修改ssh端口后，git client处的处理方法###

在linux上有两种方法。

  1. 修改git目录下的.git/config文件。
    找到url参数，把url参数从git协议改成指定端口的ssh协议。如：url=ssh://user@gitserverip:port/home/codebase注意后面的home/ codebase一定得是主机的上git仓库所在的absolute path!。

  2. 修改~/.ssh/config，如果没有这个配置文件就创建一个，只需要填host,port两项就可以了，实例：
    如果你的git server ip是114.114.114.114 ssh port 是8080.你只需要在config上加两行, 

    host 114.114.114.114  
port 8080

###archlinux相关###

####在启动x后自动关笔记本本身的显示器，开启外接显示器####

在~/.xinitrc 里加上一条 xrandr --output LVDS1 --off, 注意要写在exec vm/de之前，比如我用的是awesome，就要在exec awesome前加上xrandr的命令

####配置awesome 自启动(autostart)软件####

编辑~/.config/awesome/rc.lua.
在最后加入awful.tuil.spawn(“applicationname”)
一次性启动多个应用的写法：

do
  local cmds = 
  { 
    "swiftfox",
    "mutt",
    "consonance",
    "linux-fetion",
    "weechat-curses",
    --and so on...
  }

  for _,i in pairs(cmds) do
    awful.util.spawn(i)
  end
end
